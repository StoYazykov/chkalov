===== ТИПЫ =====

Типы: Int, Long, Float, Double, Char, String, UNSInt, UNSLong, UNSFloat, UNSDouble, UNSChar. Думаю, что беззнаковая строка никому не нужна.

===== ПЕРЕМЕННЫЕ =====

Статически типизированная переменная: var имя=значение или var: Тип имя=значение.
(код)
var a=5252
или
var: Int a=5252

А внутри компилятора происходит это:

(вырезка из кода компилятора)
scopes.addVar("a");
(scopes.getVar("a").value)=val.value;
(scopes.getVar("a").type)=INT;

И при запуске программы ВМ видит это:
(ВМ)
ipush 5252
istore 0

-------------

Динамически типизированная переменная: dyn имя=значение. Динамическая переменная ИМЕЕТ ТИП Dyn, похожий на Any из Котлина, НА УРОВНЕ КОДА.
На уровне компилятора же она имеет один из основных типов - в зависимости от того, какое значение она в данный момент хранит. На уровне ВМ всё есть Int/Long число (притом, стек хранит 4-байтовые Int'овые числа - просто
для 8-байтового Long'a задействуется две ячейки стека, а Float'ы преобразуются в Int'ы. Double'и, естественно, в два Int'а.

(код)
dyn a=5 // сейчас для КОМПИЛЯТОРА тип Int
a="s" // тип можно поменять! Для КОМПИЛЯТОРА он сейчас String.

(ВМ)
ipush 5
istore 0
spush "s"
istore 0

===== ФУНКЦИИ =====

Функция ничего не возвращающая без аргументов: fun ИмяФункции { тело функции }  (да, если нет аргументов - пустые скобки не нужны! Урррааа!!!)
Функция ничего не возвращающая с аргументами: fun ИмяФункции(аргументы, ещёаргументы) { тело функции }
Функция возвращающая что-нибудь без аргументов: fun ИмяФункции: ТипТогоЧтоВозвращаем { тело функции }
Функция возвращающая что-нибудь с аргументами: fun ИмяФункции: ТипТогоЧтоВозвращаем(аргументы, ещёаргументы) { тело функции }
